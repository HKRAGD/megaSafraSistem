Estamos com um problema de fluxo no sistema FSM de produtos. Quando um ADMIN solicita a saída de um produto através do formulário de movimentação (opção "mover" na página de produtos), o produto está sendo retirado diretamente ao invés de ir para o status "AGUARDANDO_RETIRADA" para o OPERATOR confirmar.

O fluxo correto deveria ser:
1. ADMIN solicita saída através do formulário de movimentação → produto fica "AGUARDANDO_RETIRADA"  
2. OPERATOR confirma a retirada física → produto fica "RETIRADO"

Mas está acontecendo:
1. ADMIN solicita saída → produto vai direto para "RETIRADO" 

Preciso que você analise:
1. Como funciona atualmente o fluxo de solicitação de saída
2. Onde está o problema na implementação 
3. Como corrigir para seguir o FSM correto
4. Se há confusão entre "partialExit" (saída direta) vs "requestWithdrawal" (solicitação de saída)
5. Como o formulário de movimentação deveria chamar a função correta

O sistema tem roles ADMIN/OPERATOR com FSM bem definido onde ADMIN solicita e OPERATOR executa fisicamente.
